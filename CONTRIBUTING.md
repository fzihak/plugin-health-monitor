# Contributing to Health Radar

First off, thank you for considering contributing to Health Radar! It's people like you that make Health Radar such a great tool.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for Health Radar. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

- Use the GitHub **Bug Report** template.
- Provide clear and reproducible steps.
- Describe the exact PHP and WordPress version you are using.

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion for Health Radar, including completely new features and minor improvements to existing functionality.

- Use the GitHub **Feature Request** template.
- Be clear about why this feature would be useful to most users.

### Pull Requests
- **Create a new branch** for your feature or bug fix: `git checkout -b feature/your-feature-name`.
- **Follow WordPress Coding Standards**: We run PHP_CodeSniffer automatically on ALL pull requests. Ensure your code complies with the `WordPress` ruleset.
- **Write descriptive commit messages**.
- **Update documentation** if you are changing functionality.
- Open a Pull Request and fill out the provided template.

## Coding Standards

- PHP code must follow the [WordPress Coding Standards](https://make.wordpress.org/core/handbook/best-practices/coding-standards/php/).
- JavaScript should follow the [WordPress JavaScript Coding Standards](https://make.wordpress.org/core/handbook/best-practices/coding-standards/javascript/).
- CSS should follow the [WordPress CSS Coding Standards](https://make.wordpress.org/core/handbook/best-practices/coding-standards/css/).
